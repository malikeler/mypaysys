系统支持的到账通知

  • 首页
  • 到账通知

  • API 主动查询

    • 实时查询订单状态，获取最新支付结果

    • 回调异常时可主动查单，避免订单遗漏。

    • 网络波动或系统异常仍可确认交易状态。

    • 可按设定频率自动查询订单信息

    • 订单状态及时更新，提高业务处理效率。


  • 短信通知（SMS）

    • 短信通知适用范围广，可覆盖未安装支付应用或未开启推送服务的用户群体。

    • 用户无需连接移动网络或 Wi-Fi，只要手机具备短信功能即可正常接收通知。

    • 短信发送按条计费，大批量通知会增加运营成本，适合重点业务场景使用。

    • 常与Webhook回调、API查询或 App Push配合使用，作为支付到账提醒和异常通知的补充方案。


  • APP Push 通知

    • 支付完成后可即时向用户推送通知，快速反馈交易结果，提高信息传递效率。

    • 通知直接显示在手机界面，无需主动查询订单状态，操作更加便捷流畅。

    • 用户需提前安装对应应用并开启通知权限，才能正常接收推送消息。

    • 支持快速触达用户，满足电商、游戏、会员充值等高频支付业务需求。

    • 可展示订单金额、交易状态、到账时间等详细信息，提升信息可读性。

    • 广泛应用于电子钱包、移动支付及金融科技平台的支付通知场景。


  • 邮件通知（Email）

    • 用于标准化账务核对与差异检查，方便财务对账处理

    • 支持数据归档存储，便于后续审计与历史查询。

    • 按周期批量生成（如日/月），不依赖实时传输。
